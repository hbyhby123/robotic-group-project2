## 3 实验1：巡光

> **任务：** 小车跟随手电筒光斑自主转向行进。

### 3.1 硬件 —— 光敏电阻分压电路

光敏电阻的核心特性：**光越强，阻值越小**。左右各一个，通过分压电路将光照转为电压。

| 项目 | 说明 |
|------|------|
| 电路类型 | `VCC → 10kΩ → OUT → LDR → GND`（LDR 接地端） |
| 左传感器 | OUT 接 **A1**，供电由引脚 7 控制 |
| 右传感器 | OUT 接 **A2**，供电由引脚 8 控制 |
| 输出范围 | `analogRead()` 返回 0~1023 |

**工作原理（直觉理解）：**
- 🔆 光照强 → LDR 阻值 ↓ → OUT 电压 ↓ → 读数 ↓
- 🌑 光照弱 → LDR 阻值 ↑ → OUT 电压 ↑ → 读数 ↑

> **为什么 LDR 接在接地端？** 强光→读数小，弱光→读数大，符合直觉，串口调试时一眼就能判断。反过来接（LDR 接电源端）读数方向相反，调试时容易混淆。

**一句话总结：** 利用 10kΩ 固定电阻和光敏电阻组成分压电路，Arduino 通过 `analogRead` 读取 OUT 电压来判断光照强弱。

**测试方法：** 手电筒分别照左右两侧——照射侧读数明显下降，另一侧保持高位。用万用表验证 $V_{OUT} = VCC \times \frac{R_{LDR}}{R_{LDR} + 10kΩ}$。

### 3.2 软件 —— 优化版 PD 控制

**所需库：** 无外部库，全部使用 Arduino 内置函数。

**设计思路：** 比较左右光敏电阻读数，哪边光强就往哪边转。

**信号处理链：** 原始读数 → 低通滤波 → 误差计算 → 多级死区 → PD 计算 → 动态限幅 → 速率限制 → 差速驱动

**低通滤波：** 光敏电阻存在噪声，先做指数移动平均（α=0.40）。

**误差计算：** error = **右 − 左**。右侧读数更大说明右边更暗（光在左边），需要左转。

- error > 0 → 光在左边 → **左转**（左轮减速、右轮加速）
- error < 0 → 光在右边 → **右转**

**多级死区 + 动态限幅：**

| 机制 | 数值 | 作用 |
|------|:--:|------|
| 误差死区 `deadBand` | 45 | |error|<45 视为居中，置 0 |
| 直线锁定 `straightBand` | 56 | |error|<56 时 PID 全清零，彻底不修正 |
| 输出死区 `outputDeadBand` | 9 | |correction|<9 时输出置 0 |
| 小偏差限幅 | 20 | 直线时修正上限，防摆尾 |
| 大偏差限幅 | 42 | 弯道时放宽上限 |
| 变化速率限制 | 4.2/轮 | 防止突然甩头 |

**PID 参数：**

| 参数 | 值 | 说明 |
|------|:--:|------|
| Kp | 0.20 | 小 P 避免直线摆尾 |
| Kd | 0.055 | 微分抑制振荡 |
| Ki | 0 | 不使用（光源持续运动，无稳态偏差需要累积） |
| 基础速度 L/R | 85/81 | 非对称补偿电机个体差异 |
| 速度范围 | 25~115 | 最低不卡死，最高不冲过头 |
| 控制频率 | ≈83Hz | 12ms 采样周期 |

**PID 调参效果：**

| 参数 | 过大 | 过小 |
|------|------|------|
| P | 左右摇摆、蛇形前进，严重时原地打转 | 转向迟钝，跟不上光斑 |
| D | 转弯"肉"，光源移动后迟迟不响应 | 抑制不足，冲过头来回震荡 |
| I | 积分饱和，转弯滞后甚至失控 | —（不使用） |

**核心代码：**

```cpp
// —— 引脚：A1左 A2右，L298N六线(ENA=9,ENB=10,IN1~4=2,3,4,5) ——

void readSensors() {
    filteredLeft  = 0.40 * filteredLeft  + 0.60 * analogRead(A1);
    filteredRight = 0.40 * filteredRight + 0.60 * analogRead(A2);
}

void calculatePID() {
    error = filteredRight - filteredLeft;
    if (abs(error) < 45) error = 0;                          // 死区
    if (abs(error) < 56) { error=0; integral=0; derivative=0; } // 直线锁定

    derivative = error - lastError;
    rawCorrection = 0.20 * error + 0.055 * derivative;
    int maxC = (abs(error) > 180) ? 42 : 20;                 // 动态限幅
    rawCorrection = constrain(rawCorrection, -maxC, maxC);
    if (abs(rawCorrection) < 9) rawCorrection = 0;           // 输出死区

    // 变化速率限制
    float diff = rawCorrection - lastCorrection;
    if (diff > 4.2)      correction = lastCorrection + 4.2;
    else if (diff < -4.2) correction = lastCorrection - 4.2;
    else                  correction = rawCorrection;
    lastCorrection = correction; lastError = error;
}

void driveMotors() {
    leftSpeed  = constrain(85 - correction, 25, 115);   // 非对称基础速度
    rightSpeed = constrain(81 + correction, 25, 115);
    // digitalWrite方向 + analogWrite PWM → L298N六线模式
}
```

### 3.3 最终性能

| 方面 | 评价 |
|------|------|
| ✅ 优势 | 低通滤波 + 多级死区使直线非常平稳、不"哆嗦"；动态限幅弯道自动放开；非对称基础速度补偿电机差异 |
| ❌ 不足 | 环境光强时两个 LDR 基底读数同时压低，差值被稀释，偶有误判 |
| 🔍 原因 | 无环境光补偿——两路信号叠加相同环境光分量，目标光源差异被"稀释" |

## 4 实验2：避障

> **任务：** 小车检测障碍物，选择开阔方向自主前进。

### 4.1 硬件 —— HC-SR04 超声波传感器 + 舵机

| 项目 | 说明 |
|------|------|
| 工作原理 | 发射 40kHz 超声波 → 接收回波 → $distance = duration \times 0.034 / 2$ cm |
| 量程 | 约 2cm~400cm，`pulseIn` 超时 38ms（≈6.5m），超时返回 999cm |
| 扫描方式 | 舵机（引脚 6）旋转扫三个角度：**120°（左前）、90°（正中）、60°（右前）** |
| 驱动模式 | L298N **四线模式**，ML1/2=2,3、MR1/2=4,5，无 PWM |

**为什么避障用四线模式？** 避障要求检测到障碍后迅速后退并转向，对电机**瞬时扭矩**要求高。四线模式下 ENA/ENB 由 jumper 直连高电平，电机始终最大功率运行，扭矩充足。相比六线模式接线更简单，且避障不涉及巡线那样的精细轮速差调节。

**为什么只能控制运动时间？** 四线模式下无法 `analogWrite()` 输出 PWM 调速，电机只有全速/停止。控制移动距离只能靠调节运转**持续时间**——`T_FWD`（180ms）、`T_FWD_SLOW`（120ms）、`T_BCK`（150ms）。

**测试方法：** 舵机固定 90°，前方 30cm 放障碍物，串口打印测距值，误差 < 1cm。

> 📷 **插图 5：** 超声波 + 舵机前向安装，标注 60°/90°/120° 扫描角度

### 4.2 软件 —— 四级距离决策 + 三重防抖

**所需库：** `<Servo.h>`（Arduino 内置）。

**可靠测距：** `pulseIn(ECHO, HIGH, 38000)` 超时 38ms（≈6.5m），超时返回 999cm 防卡死。

**扫描策略：** 对角线前向扫描（60°/90°/120°），选前向斜角而非正侧方——前向信息更早更有用，侧方墙壁不会误导转向。

**四级距离决策：**

```
>150cm             70~150cm          30~70cm           <30cm
 CRUISE              CLEAR            WARNING          CRITICAL
 "开阔"             "比较开阔"        "有点近"         "太近!"
   │                   │                 │                 │
   ▼                   ▼                 ▼                 ▼
放心直行           正常直行         对角线更开阔?      后退→重扫
重置方向惯性       可微调航向        →有：后退转向      →转弯→强制前进
                                     →无：慢速前进
```

**直行时长动态调整：** 前方 >1m 直行更久，<1m 直行更短，避免狭窄空间冲过头。

| 层级 | 距离 | 行为 | 直行时长 |
|:----:|------|------|:--:|
| CRUISE | >150cm | 全速直行，重置方向偏好 | 180ms（长） |
| CLEAR | 70~150cm | 直行 + 可按偏好微调 | 180ms |
| WARNING | 30~70cm | 对角线更开阔则转向，否则慢行 | 120ms（短） |
| CRITICAL | <30cm | 后退→重扫→转弯→**强制直行** | — |

**三重防抖：**

① **小步进（60ms/步）：** 每步重测前方，发现障碍立刻停。

② **方向惯性（20cm 滞回）：** 上次左转则需右方领先 20cm+ 才换向，防止左右反复摇摆。

③ **转弯后强制前进（mustGoFwd）：** 每次转弯后强制往前走一小段，离开原地后再重新评估——这是杜绝原地打转的核心设计。

**受困处理：** 三方向均 <30cm 时额外后退一次再择向突围。

**核心代码：**

```cpp
#include <Servo.h>
Servo servo;  // 引脚 6
const int TRIG=11, ECHO=12, ML1=2,ML2=3,MR1=4,MR2=5;
const int CRITICAL=30, WARNING=70, T_STEP=60;

float getDist() {
    digitalWrite(TRIG,LOW); delayMicroseconds(2);
    digitalWrite(TRIG,HIGH); delayMicroseconds(10);
    digitalWrite(TRIG,LOW);
    long dur = pulseIn(ECHO, HIGH, 38000);
    return (dur==0) ? 999 : dur/58.0;
}

void scanDiagonals() {
    servo.write(120); delay(100); distL = getDist();   // 左前
    servo.write(90);  delay(100);
    servo.write(60);  delay(100); distR = getDist();   // 右前
    servo.write(90);  delay(100);                      // 归中
}

int chooseDir(int left, int right) {
    if (lastTurn == 0) return (left > right) ? -1 : 1;
    if (lastTurn == -1) return (right > left + 20) ? 1 : -1;
    else                return (left > right + 20) ? -1 : 1;
}

void fwdStepped(int totalMs) {   // 小步进前进，途中检测
    int steps = totalMs / T_STEP;
    for (int i=0; i<steps; i++) {
        if (quickF() < CRITICAL) { halt(); return; }
        goFwd(); delay(T_STEP);
    }
    halt();
}

void loop() {
    distF = quickF();
    if (distF < WARNING || cycleCount%5==0) scanDiagonals();

    if (mustGoFwd && distF > CRITICAL) {          // 转弯后强制前进
        fwdStepped(120); mustGoFwd=false; return;
    }
    mustGoFwd = false;

    if (distF > 150) {                              // CRUISE
        lastTurn=0; fwdStepped(180);
    } else if (distF < CRITICAL) {                  // CRITICAL
        goBck(); delay(150); halt(); scanDiagonals();
        if (distL<CRITICAL && distR<CRITICAL) { goBck(); delay(150); halt(); scanDiagonals(); }
        int dir = chooseDir(distL, distR);
        turnChunked(dir==-1, 130); lastTurn=dir; mustGoFwd=true;
    } else if (distF < WARNING) {                   // WARNING
        bool L=(distL>distF+15), R=(distR>distF+15);
        if (L||R) { goBck(); delay(80); halt();
            turnChunked(chooseDir(distL,distR)==-1, 100);
            lastTurn=chooseDir(distL,distR); mustGoFwd=true; }
        else { fwdStepped(120); }
    } else { fwdStepped(180); }                     // CLEAR
    cycleCount++;
}
```

## 5 实验3：巡线

> **任务：** 小车沿黑线轨迹稳定行驶，能通过直角弯。

### 5.1 硬件 —— TCRT5000 红外传感器 + ADS1115

三个 TCRT5000 红外反射传感器安装在车底前下方，通过 **ADS1115** 16 位 ADC 高精度采集。

| 项目 | 说明 |
|------|------|
| 传感器型号 | TCRT5000（内置红外发射管+光电晶体管+LM393 比较器）×3 |
| 理论检测距离 | 2~30mm（数据手册） |
| 安装布局 | **非对称**——左右在基准位置，**中间前移约 3cm**，提前感知弯道 |
| 信号采集 | **ADS1115**（16位，I2C 0x48，±6.144V，860 SPS） |
| 驱动模式 | L298N 六线模式，ENA=10、ENB=9，IN1~4=4,5,2,3 |

**为什么加 ADS1115？** Arduino 内置 ADC 仅 10 位，ADS1115 16 位分辨率高 64 倍，能稳定区分黑白地面几十 mV 的电压差。

**工作原理：** 红外光 → 地面反射 → 接收管。黑色吸光（反射弱→输出电压高），白色反光强（输出低）。传感器在**黑线上方时读数高**。

**特性化实验发现（来自独立 TCRT5000 实验报告）：**

| 子实验 | 核心发现 | 工程启示 |
|--------|----------|----------|
| 模块功能测试 | 三个模块存在个体灵敏度差异（制造公差） | 必须逐个标定阈值 |
| 电位器方向影响 | 顺时针灵敏度增大；过度调高引入噪声误触 | 调到"刚好够用" |
| 多点距离标定 | 距离越近输出越小；**极近（<1.5cm）出现非线性反转** | 安装高度不能太低 |
| 黑胶带反射阈值 | **临界距离约 1.55cm**，比较器闪烁；0.05cm 时黑胶带吸光过强 | 验证黑白区分原理 |

> **安装高度选定：** 太近（<1cm）读数异常，太远（>2.5cm）区分度不足，最终选 **1.5~2cm**。

**筛选方法：** ⚠️ 课上发的模块很多存在质量问题。每个传感器在黑/白纸上 5mm 高度分别读 ADS1115，黑白差值过小的淘汰。5 个中筛出 3 个。

**阈值标定：** 逐个传感器 $threshold = (V_{black} + V_{white}) / 2$。顺时针和逆时针需分别标定（角度不同导致系统偏差）。最终方案：左 14500、中 15000、右 10900（ADS1115 原始值）。

### 5.2 软件 —— 混合误差 PID + 支点转弯

四个实验中算法最复杂的部分，最终方案（v5）融合了连续 ADC 和二进制状态机的优点。

**所需库：** `<Wire.h>`（内置）+ `<ADS1115_WE.h>`（库管理器安装，Wolfgang Ewald）。

**传感器布局：**

```
       [左]          [右]       ← 两侧基准
         \          /
          \        /
         [中间·超前3cm]        ← 提前感知弯道
```

**混合误差计算：** 利用 ADS1115 原始 ADC 值做连续误差，二进制状态做极端强纠偏。

| 状态 | `error` | 策略 |
|:----:|:-----:|------|
| B010/B110/B011 | `(rawRight-rawLeft)/140` | 连续 ADC 精细转向 |
| B100 | -95 | 硬编码强左转 |
| B001 | +95 | 硬编码强右转 |
| B000 | ±110 交替搜索 | 400ms 间隔反转方向 |

**支点转弯（Pivot Turn）—— 过直角弯核心：**

| 状态 | 左轮 | 右轮 | 效果 |
|:----:|:--:|:--:|------|
| B011（左离线） | 70 | 0 | 绕右轮右转 |
| B110（右离线） | 0 | 70 | 绕左轮左转 |
| B001（仅右在线） | 60 | -95 | 左进右退原地 pivot |
| B100（仅左在线） | -95 | 60 | 左退右进原地 pivot |

**中间传感器超前预警减速：** 中间读到 0（白）→ 前方即将偏离 → **降速 ≤35**。丢线状态不触发（需要速度做搜索）。

**直道稳定：** B101/B111 连续第 3 次起清零 derivative 和 integral，消除弯道→直道的甩动。

**丢线搜索 + 恢复冷却：** 400ms 交替反转搜索方向；找回线后 300ms 冷却期防再次丢线。

**积分过零清零：** error 变号时旧方向积分拖累新方向，必须清零。大跳变时积分减半。

**PID 参数：**

| 参数 | 值 | 说明 |
|------|:--:|------|
| Kp | 0.70 | 中等 P |
| Ki | 0.015 | 微量 I，帮助 S 弯持续转向 |
| Kd | 2.5 | 大 D 主导，对状态变化极度敏感 |
| 积分限幅 | ±300 | 防饱和 |

**速度自适应：**

| 误差 | ≥110 | ≥80 | ≥40 | <40 | 中间=0 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| 基础速度 | 40 | 50 | 50 | 90 | ≤35 |

**调试要点：**
- 重点观察**串口监视器中的 switch 状态**，根据其变化调整 threshold
- threshold **不能过大**（直角弯不灵敏），也**不能过小**（卡顿或不动）
- 速度越慢巡线越稳，但速度是评分项，需在**鲁棒性和速度间寻求平衡**

**核心代码：**

```cpp
#include <Wire.h>
#include <ADS1115_WE.h>
ADS1115_WE adc(0x48);
// 引脚：ENA=10,ENB=9,IN1~4=4,5,2,3
// 阈值：左14500 中15000 右10900

void Scan() {
    irSensors = B000;
    rawLeft=readADS(2); rawMid=readADS(1); rawRight=readADS(0);
    int raw[3]={rawLeft,rawMid,rawRight};
    for(int i=0;i<3;i++) {
        irSensorDigital[i]=(raw[i]>=thresholds[i])?1:0;
        irSensors|=(irSensorDigital[i]<<(2-i));
    }
}

void CalculateError() {
    int contErr = constrain((rawRight-rawLeft)/140, -110, 110);
    switch(irSensors) {
        case B010:case B110:case B011: error=contErr; break;
        case B101:case B111:
            error=(consecutiveB111<=2)?((abs(lastError)<40)?contErr:lastError):contErr;
            break;
        case B100: error=-95; break;   case B001: error=95; break;
        case B000:
            error=b000Active?b000SearchDir*110:((lastError<0)?-110:110);
            break;
        default: error=contErr;
    }
}

void PID_Control() {
    derivative=constrain(error-lastError, -30, 30);
    if(irSensors==B101||irSensors==B111) {
        consecutiveB111++;
        if(consecutiveB111==3){ derivative=0; integral=0; }
    } else consecutiveB111=0;
    // B000搜索+冷却...
    // 积分过零清零...
    int spd=(abs(error)>=110)?40:(abs(error)>=80)?50:(abs(error)>=40)?50:90;
    if(irSensorDigital[1]==0&&irSensors!=B000) spd=min(spd,35);
    pid=0.70*error+0.015*integral+2.5*derivative; lastError=error;
    leftMotorSpeed=constrain(spd+pid,-255,255);
    rightMotorSpeed=constrain(spd-pid,-255,255);
    // 支点转弯硬编码覆盖...
}

void Drive(int L,int R) {
    if(abs(R)<8) analogWrite(enablePin1,0);
    else if(R>0){ digitalWrite(in1,HIGH);digitalWrite(in2,LOW);analogWrite(enablePin1,R); }
    else        { digitalWrite(in1,LOW); digitalWrite(in2,HIGH);analogWrite(enablePin1,-R); }
    // 左电机同理...
}
```

### 6.3 软件 —— 三态状态机

**所需库：** `<Wire.h>`（内置）+ `<Adafruit_Sensor.h>` + `<Adafruit_BME280.h>`（库管理器安装，Adafruit）。

**三态状态机：**

```
                  ┌──────────────┐
        Z > 1.4g  │   静止模式    │  Z < 0.6g
      ┌─────────→ │  (车轮停转)   │ ←─────────┐
      │           └──────┬───────┘            │
      │                  │ Z > 1.4g            │
      │           ┌──────▼───────┐            │
      │           │   抬升模式    │            │
      │   Z > 1.4g│  (车轮转动)   │ Z < 0.6g   │
      └────────── │               │ ──────────┘
                  └───────────────┘
```

- **Z > 1.4g：** 静止时 Z≈0（减去重力后），抬升瞬间向上加速度叠加
- **Z < 0.6g：** 放下时短暂失重，Z 骤降
- **2 秒冷却：** 模式切换后 2 秒内忽略变化，防抖动误触发

**数据输出（每 500ms，传感器更新 ≥1次/秒，符合评分要求）：**

```
[BME280] 温度：28.5°C，湿度：55%，气压：1000.0 hPa
[MPU6050] 加速度计：X=0.12g，Y=0.08g，Z=1.02g | 陀螺仪：X=1.5°/s，Y=-0.8°/s，Z=0.3°/s
---
```

**I2C 传感器功能（对照评分标准，10 分）：**

| 评分项 | 分值 | 达成 |
|--------|:--:|:--:|
| 同时连接 BME280 和 MPU6050 | 2 | ✅ I2C 双设备，地址 0x76+0x68 无冲突 |
| 读取 BME280 并标注格式化 | 2 | ✅ 温湿压标注输出，500ms/次 |
| 读取 MPU6050 并标注格式化 | 2 | ✅ 加速度+陀螺仪分项标注，含单位 |
| 快速抬起→车轮转动并保持 | 2 | ✅ Z>1.4g → LIFT → motor(true) |
| 快速放下→车轮停止并保持 | 2 | ✅ Z<0.6g → PUTDOWN → motor(false) |

**核心代码：**

```cpp
#include <Wire.h>
#include <Adafruit_Sensor.h>
#include <Adafruit_BME280.h>
Adafruit_BME280 bme;
#define MPU6050_ADDR 0x68
// 电机：ENA=10,ENB=9,IN1~4=4,5,2,3

// —— 底层 I2C 读写 ——
void writeReg(uint8_t addr, uint8_t reg, uint8_t val) {
    Wire.beginTransmission(addr); Wire.write(reg);
    Wire.write(val); Wire.endTransmission();
}
void readRegs(uint8_t addr, uint8_t reg, uint8_t* buf, uint8_t len) {
    Wire.beginTransmission(addr); Wire.write(reg);
    Wire.endTransmission(false);
    Wire.requestFrom(addr, len);
    for (uint8_t i=0; i<len; i++) buf[i]=Wire.read();
}

// —— 加速度读取（±2g，已校准） ——
void readAcc(float &ax, float &ay, float &az) {
    uint8_t d[6]; readRegs(MPU6050_ADDR, 0x3B, d, 6);
    ax = ((int16_t)((d[0]<<8)|d[1]))/16384.0 - accOffsetX;
    ay = ((int16_t)((d[2]<<8)|d[3]))/16384.0 - accOffsetY;
    az = ((int16_t)((d[4]<<8)|d[5]))/16384.0 - accOffsetZ;
}

// —— 陀螺仪（±250°/s，低通+死区） ——
void readGyro(float &gx, float &gy, float &gz) {
    uint8_t d[6]; readRegs(MPU6050_ADDR, 0x43, d, 6);
    float rx=((int16_t)((d[0]<<8)|d[1]))/131.0 - gyroOffsetX;
    float ry=((int16_t)((d[2]<<8)|d[3]))/131.0 - gyroOffsetY;
    float rz=((int16_t)((d[4]<<8)|d[5]))/131.0 - gyroOffsetZ;
    gyroFiltX=0.15*rx+0.85*gyroFiltX; gx=(abs(gyroFiltX)<0.5)?0:gyroFiltX;
    gyroFiltY=0.15*ry+0.85*gyroFiltY; gy=(abs(gyroFiltY)<0.5)?0:gyroFiltY;
    gyroFiltZ=0.15*rz+0.85*gyroFiltZ; gz=(abs(gyroFiltZ)<0.5)?0:gyroFiltZ;
}

// —— 200点静止校准 ——
void calibrateMPU6050() {
    const int N=200; float sAx=0,sAy=0,sAz=0,sGx=0,sGy=0,sGz=0;
    for(int i=0;i<N;i++) {
        float ax,ay,az,gx,gy,gz;
        readAccRaw(ax,ay,az); readGyroRaw(gx,gy,gz);
        sAx+=ax;sAy+=ay;sAz+=az; sGx+=gx;sGy+=gy;sGz+=gz; delay(5);
    }
    accOffsetX=sAx/N; accOffsetY=sAy/N; accOffsetZ=sAz/N - 1.0;
    gyroOffsetX=sGx/N; gyroOffsetY=sGy/N; gyroOffsetZ=sGz/N;
}

// —— 状态机（检测50ms周期，冷却2s） ——
enum State { STATIONARY, LIFTED, PUTDOWN };
State state = STATIONARY;

void loop() {
    if (millis()-lastDet >= 50) {
        float ax,ay,az; readAcc(ax,ay,az);
        if (millis()-lastSwitch >= 2000) {
            switch(state) {
                case STATIONARY: if(az>1.4){state=LIFTED;lastSwitch=millis();} break;
                case LIFTED:
                    if(az<0.6){state=PUTDOWN;lastSwitch=millis();}
                    else if(az>1.4) lastSwitch=millis(); break;
                case PUTDOWN:
                    if(az>1.4){state=LIFTED;lastSwitch=millis();}
                    else if(az<0.6) lastSwitch=millis(); break;
            }
        }
        lastDet=millis();
    }
    motor(state == LIFTED);

    if (millis()-lastDisp >= 500) {
        Serial.print("[BME280] 温度："); Serial.print(bme.readTemperature(),1);
        Serial.print("°C，湿度："); Serial.print(bme.readHumidity(),1);
        Serial.print("%，气压："); Serial.print(bme.readPressure()/100.0,1);
        Serial.println(" hPa");
        float ax,ay,az,gx,gy,gz; readAcc(ax,ay,az); readGyro(gx,gy,gz);
        Serial.print("[MPU6050] 加速度计：X="); Serial.print(ax,2);
        Serial.print("g，Y="); Serial.print(ay,2); Serial.print("g，Z="); Serial.print(az,2);
        Serial.print("g | 陀螺仪：X="); Serial.print(gx,1);
        Serial.print("°/s，Y="); Serial.print(gy,1);
        Serial.print("°/s，Z="); Serial.print(gz,1); Serial.println("°/s\n---");
        lastDisp=millis();
    }
}

void motor(bool run) {
    if(run){ digitalWrite(in1,HIGH);digitalWrite(in2,LOW);analogWrite(ENA,100);
             digitalWrite(in3,HIGH);digitalWrite(in4,LOW);analogWrite(ENB,100); }
    else   { digitalWrite(in1,LOW);digitalWrite(in2,LOW);analogWrite(ENA,0);
             digitalWrite(in3,LOW);digitalWrite(in4,LOW);analogWrite(ENB,0); }
}
```

## 7 结论与展望

### 7.1 项目总结

本项目成功设计并实现了一台能完成巡光、避障、巡线和 I2C 多传感器综合应用的小型自主移动机器人，四项任务基本功能全部实现。

**核心技术收获：**

1. **传感器标定决定系统上限。** 红外逐个阈值标定、光敏电阻选接地端、MPU6050 200 点校准、TCRT5000 特性化实验揭示的极近距离非线性——前期标定工作直接影响最终表现。

2. **PID 调参是系统化过程。** 理解每个参数的物理含义（P=现在偏多少，D=变化多快，I=历史累积量），逐轮记录参数和表现，能清晰判断该调大还是调小。

3. **工程细节往往是瓶颈。** 电机接线脱落、传感器质量筛选、I2C 焊接虚焊、电池接触不良——这些"非算法问题"占用了大量调试时间，但它们是真实工程能力的一部分。

4. **串口监视器是调试的眼睛。** 实时看传感器原始值和状态切换，能快速定位问题是"读不准"、"算不对"还是"动不了"。

5. **最简单的设计往往最可靠。** 巡线的支点转弯（一轮刹停）、避障的强制前进（一个 bool 标志）、巡光的直线锁定（|error|<56 清零 PID）——核心代码都很短，思路对了才写得出来。

### 7.2 未来改进
